异步任务的初步理解

首先, javascript 代码是单线程的, 但是 js 中又有异步任务, 那么 js 是如何实现异步任务的呢

  • 先来看看同步任务

    js 中的代码都是在 栈 中执行的, 这很好理解, 符合单线程的规定
    所以同步任务就是直接放入栈中运行, 一个任务没有完成就不会去执行另外一个任务

  • 再来看看如何实现异步任务

    对于同步任务是直接放入栈中执行
    而如果一个任务是异步任务
    那么它将会放入另一个一个地方(队列) 等待 执行

    不过注意, 异步任务只是放入队列中等待, 并不是在队列中执行
    只有当栈中的任务清空了, 系统才会去查看队列中是否有任务需要执行调用
    所以异步任务本质还是都栈中执行(这也符合单线程的要求嘛)

  • 说一下同步任务和异步任务的一个区别

    同步任务直接放入栈中执行
    所以对于同步任务的执行顺序可以直接利用栈的知识来理解

    而异步任务虽然也是放入栈中执行
    但请注意, 异步任务是先在队列中等待
    之后是一个一个进入栈中执行
    而不是全部一次性进入
    这才符合队列的(先进先执行的规范嘛)

  • 补充一下所谓 js 的单线程执行

    刚开始理解时可能都会觉得为什么要使用单线程呢, 多线程不是更快吗？

    如果你也有这样的想法的话
    你要清楚一点, js 的单线程指的的代码单线程
    但这并不代码 js 不能利用多线程
    一个经典的理解就是定时器

    请看下面的代码

    ;(() => {
  for (let i = 0; i < 3000; i++) console.log('1')
  setTimeout(() => { console.log('ddddd'); }, 3000);
})();

    如果真是单核执行, 那么但输出全部的 1 后应该再等 3 秒才输出 ‘ddddd’
    但结果却是输出1后直接就输出 ddddd 了
    这说明什么？
    说明这个 3000 时间不是等到 for 循环执行后采取计算的
    而是代码一开始执行的时候就开始计算了
    也就是说计算这个 3s 的时间是和执行 for 循环一起的
    但是因为 js 代码 是单线程的
    所以代码中的输出 ‘ddddd’ 必须等到 for 循环执行完成后才会输出

异步任务中也有优先级

消息队列和作业队列

先来看看下面这段代码

setTimeout(() => { console.log('2'); }, 0);
Promise.resolve().then(() => { console.log('1'); })
console.log('同步任务最先调用');

你会发现 then 代码比 setTimeout 代码更早输出
但明明 then 和 setTimeout 都是异步任务
如果你知道了异步任务中也有优先级那就很好理解了

setTimeout 代码会在 消息队列中等待调用
而 Promise(即 then ) 会在作业队列中等待调用
而系统规定了作业队列的优先级比消息队列的优先级高

所以当栈清空时(输出了’同步任务最先调用’后)
系统会优先去调用作业队列中的任务
当作业队列中的任务全部执行完成后才会去调用消息队列中的任务
这就能解释上面代码输出顺序的原因了
